PART ONE
AI是怎么工作的？
（通俗版）
不需要懂技术，只需要建立正确预期
Token是理解AI上下文限制的关键概念。中文相对英文消耗token更多，在实际使用时要注意输入长度。

Token：AI看到的不是文字

AI不直接读取文字，而是把语言切成碎片（Token）再处理

  • 英文约 3-4个字母 = 1 Token
  • 中文约 1.5-2个汉字 = 1 Token
  • 标点、空格也占 Token
GPT-4 上下文窗口约 128K tokens
≈ 一本中篇小说的字数
Claude 3.5 可达 200K tokens
示例：「兖矿资产管理集团」的Token切分
兖矿 资产 管理 集团
示例：「请分析这家公司的商业模式」
请 分析 这家 公司 的 商业 模式
每次对话，AI会将完整上下文（历史对话+你的问题）一并处理。超出限制的内容会被截断，这是为什么长对话有时"失忆"的原因。
用"预测下一个词"来解释LLM的核心机制。这个类比非常直观，有助于理解后面讲的幻觉问题。

本质上，LLM是人类知识的压缩与重组——它从未真正"理解"，但能极其精确地预测。
这既是它强大的原因，也是它会犯错的根源。
注意力机制是Transformer架构的核心创新。用"划重点"的类比非常好理解，不需要深入技术细节。

注意力机制：AI如何"抓重点"

当AI处理一段文字时，它会为每个词分配不同的"注意力权重"

这家公司的技术壁垒很强，但 客户集中度风险 需要关注——尤其是在 政策环境 变化的背景下
🟡 深色高亮 = 高注意力权重   🔵 浅色高亮 = 中等注意力权重
📌 类比
就像分析师读报告时自动在关键句下划线，AI在处理每个词时都在实时计算：哪些上下文词对理解"当前词"最重要？
💼 对我们的意义
这意味着AI能理解长文档中的逻辑关系，不会只看字面意思。给它一份BP，它能找到前后矛盾的数据。
这一页非常重要。幻觉是很多人对AI产生误判的核心原因。必须让大家记住：数字、引用、具体案例，一定要核实。

为什么AI会"编答案"（幻觉）

⚠️ AI幻觉的本质
  • AI生成每个词时，选的是概率最高的词，而非"正确的词"
  • 训练中没有教会它说"我不知道"——它只会生成听起来合理的内容
  • 高风险区域：具体数字、文献引用、特定案例、近期事件
典型案例：AI引用的论文标题、作者、期刊均真实存在，但那篇论文实际上从未被写出来。
✅ 如何应对
  • 永远核实具体数字、市场规模、公司信息
  • 在Prompt中要求AI标注不确定的部分
    （"如果你不确定，请明确说明"）
  • 提供原始文件让AI分析，而非让它凭记忆作答
  • 将AI用于框架构建和逻辑整理，而非信息查询
